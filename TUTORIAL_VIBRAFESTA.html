Agregar fotos desde el celular

Todo se hace desde el navegador, sin apps extra
💡 Vamos a subir las fotos directamente a GitHub desde el navegador del celular. No necesitas instalar nada.
A

Comprime las fotos antes de subirlas (importante para que el sitio cargue rápido).

Abre en el navegador: squoosh.app — es gratis y funciona en el celular. Carga cada foto, cámbiala a formato WebP con calidad 80, y descárgala. Repite para todas las fotos del evento.

⚠️ Si son más de 30 fotos, es más práctico hacerlo desde computadora usando el script automático.
B

Abre GitHub en el navegador e inicia sesión con la cuenta kaimind-consulting.

Dirígete a: github.com/kaimind-consulting/VibraFesta

C

Crea la carpeta del nuevo evento.

Navega a assets → gallery. Toca Add file → Create new file. En el campo del nombre escribe:

NOMBRE_EVENTO/fotos/photos.json

Esto crea automáticamente la carpeta. En el contenido del archivo pega esto (reemplaza los nombres de foto con los tuyos):

{
  "images": [
    "foto1.webp",
    "foto2.webp",
    "..."
  ]
}

Toca Commit changes.

D

Sube las fotos a la carpeta.

Navega a assets/gallery/NOMBRE_EVENTO/fotos/. Toca Add file → Upload files. Selecciona todas las fotos WebP y toca Commit changes.

⚠️ GitHub permite subir máximo 100 archivos a la vez. Si tienes más, hazlo en tandas.
E

Sube la foto de portada.

Navega a assets/gallery/NOMBRE_EVENTO/ (un nivel arriba de fotos). Toca Add file → Upload files. Sube la foto que quieres como portada y renómbrala antes a cover.webp desde el celular.

F

Actualiza el photos.json con los nombres reales de las fotos que subiste.

Navega a assets/gallery/NOMBRE_EVENTO/fotos/photos.json. Toca el ícono del lápiz ✏️ para editar. Agrega el nombre de cada foto al listado y toca Commit changes.

G

Agrega el evento al archivo eventos.json.

Navega a assets/gallery/eventos.json. Toca el lápiz ✏️. Al inicio del archivo (después del primer [) agrega:

{
  "title": "Mayo 2026",
  "slug": "vibra-mayo-2026",
  "date": "2026-05-10",
  "cover": "assets/gallery/NOMBRE_EVENTO/cover.webp",
  "folder": "NOMBRE_EVENTO",
  "subtitle": "Edición Mayo 2026"
},

Toca Commit changes. En 2 minutos aparece en vibrafesta.com.

Actualizar las charlas del formulario

Desde el navegador del celular
1

Ve a github.com/kaimind-consulting/VibraFesta e inicia sesión.

2

Abre el archivo form_clases_vibra.html y toca el lápiz ✏️.

3

Busca el texto de la charla que quieres cambiar (usa Ctrl+F o la búsqueda del navegador) y edita el nombre/horario.

Cambia el texto dentro de <strong>...</strong> y el value="..." del checkbox correspondiente.

4

Toca Commit changes. Listo, el formulario se actualiza en 2 minutos.

Preguntas frecuentes (celular)

Q

¿Puedo hacer todo desde el celular?

Sí, pero subir muchas fotos es tedioso. Para eventos con más de 30 fotos se recomienda usar la computadora con los scripts automáticos.

Q

¿Cómo sé si el sitio se actualizó?

Espera 2 minutos después del último Commit y abre vibrafesta.com en modo incógnito.

Q

¿Dónde veo las inscripciones a las charlas?

Abre drive.google.com con la cuenta Google vinculada al formulario. Busca la hoja de cálculo de reservas.

Python — Para correr los scripts de optimización.

Verificar: abre la terminal y escribe python --version. Si aparece un número, ya lo tienes.

2

GitHub Desktop — Para subir los archivos al sitio sin usar comandos.

Descárgalo en desktop.github.com e inicia sesión con la cuenta kaimind-consulting.

3

La carpeta del proyecto en tu computadora:

C:\Users\[tu usuario]\OneDrive\Desktop\Vibra\VibraFesta
1

Agregar fotos de un evento nuevo

Hacer esto cada vez que haya un evento nuevo
A

Descarga las fotos del Drive a tu carpeta de Descargas.

Google Drive → clic derecho sobre la carpeta del evento → Descargar → quedará un ZIP en Descargas.

B

Extrae el ZIP directo en la carpeta Descargas.

Clic derecho sobre el ZIP → Extraer aquí.

C

Abre el archivo optimizar.py con el Bloc de Notas.

Lo encuentras en: VibraFesta\optimizar.py

Cambia solo estas dos líneas (las que dicen SOURCE_DIR y OUTPUT_DIR):

# Cambia solo lo que está en color claro 👇
SOURCE_DIR = r"C:/Users/[tu usuario]/Downloads/[nombre carpeta fotos]"

OUTPUT_DIR = r"C:/Users/[tu usuario]/OneDrive/Desktop/Vibra/VibraFesta/assets/gallery/[NOMBRE DEL EVENTO]/fotos"
⚠️ Importante: el nombre del evento en OUTPUT_DIR es el que aparecerá en la carpeta. Usa mayúsculas y guiones bajos, sin tildes. Ej: VIBRA_MAYO_2026
D

Corre el script de optimización.

Abre la terminal en la carpeta VibraFesta (clic derecho en la carpeta → Abrir en Terminal) y escribe:

python optimizar.py

Espera a que termine. Verás una lista de fotos procesadas. Al final dirá "¡Proceso completado!"

E

Elige una foto de portada para el evento.

Ve a la carpeta assets\gallery\[NOMBRE EVENTO]\fotos\, elige la foto que quieres como portada, cópiala y pégala un nivel arriba (en assets\gallery\[NOMBRE EVENTO]\). Renómbrala exactamente: cover.webp

F

Corre el script que genera el índice de fotos.

python test.py

Te preguntará el nombre de la carpeta. Escribe exactamente el mismo nombre que pusiste en OUTPUT_DIR (ej: VIBRA_MAYO_2026) y presiona Enter.

✅ Debe decir: "photos.json creado" con la cantidad de imágenes.
G

Agrega el evento al archivo eventos.json

Abre VibraFesta\assets\gallery\eventos.json con el Bloc de Notas.

Copia este bloque y pégalo al inicio del archivo (después del primer [), seguido de una coma:

{
  "title": "Mayo 2026",
  "slug": "vibra-mayo-2026",
  "date": "2026-05-10",
  "cover": "assets/gallery/VIBRA_MAYO_2026/cover.webp",
  "folder": "VIBRA_MAYO_2026",
  "subtitle": "Edición Mayo 2026"
},
💡 Solo cambia los valores en fondo amarillo. El resto déjalo igual.
2

Actualizar las charlas del formulario

Cuando cambien los nombres u horarios de las charlas
A

Abre el archivo VibraFesta\form_clases_vibra.html con el Bloc de Notas.

B

Busca el texto Charlas gratuitas (Ctrl+F) y edita las líneas que dicen el nombre y horario de cada charla:

<!-- Ejemplo: -->
<strong>12pm – 1pm · Charla al Ritmo del Balance</strong>
<strong>2pm – 3pm · Charla Nutrición</strong>

Cambia solo el texto entre <strong> y </strong>.

C

También actualiza el valor value="..." de cada charla para que coincida con lo que aparece en el Google Sheet:

value="12pm - 1pm · Charla al Ritmo del Balance"
3

Subir los cambios al sitio web

Hacer esto cada vez que hagas cualquier cambio
💡 El sitio web vive en GitHub. Para que los cambios aparezcan en vibrafesta.com hay que "subir" los archivos a GitHub. Eso se hace con GitHub Desktop.
1

Abre GitHub Desktop.

2

Verás una lista de archivos modificados en el panel izquierdo. Revisa que sean los correctos.

3

En el campo de abajo a la izquierda escribe un resumen corto de lo que cambiaste. Ej: "Agregar fotos Mayo 2026"

4

Clic en Commit to main → luego clic en Push origin.

✅ En 1-2 minutos los cambios estarán en vibrafesta.com. Refresca la página para verlos.

Preguntas frecuentes

Q

¿Dónde veo las inscripciones a las charlas?

Abre drive.google.com con la cuenta de Google que creó el formulario. Busca una hoja de cálculo llamada algo relacionado a "Vibra" o "Reservas". Ahí llegan todos los datos.

Q

¿Cuántas fotos puedo subir por evento?

No hay límite estricto, pero se recomienda entre 50 y 200 fotos para que la galería cargue rápido. El script las optimiza automáticamente.

Q

El script dio error, ¿qué hago?

Lo más común es que la ruta de la carpeta esté mal escrita. Verifica que el nombre de la carpeta en optimizar.py sea exactamente igual al nombre real en tu computadora (incluyendo mayúsculas y espacios).

Q

¿Cómo sé si el sitio se actualizó?

Después del Push en GitHub Desktop, espera 2 minutos y abre vibrafesta.com en modo incógnito (para evitar caché).
